## 학습 방법 (Training)

DeepSeek‑OCR는 **DeepEncoder**와 **DeepSeek‑3B‑MoE‑A570M** 디코더로 구성된 두 단계 파이프라인을 사용한다.  
전체 학습 프로세스는 다음과 같이 설계되었으며, 논문에 제시된 구체적인 하이퍼파라미터와 데이터 비율을 그대로 반영한다.

### 1️⃣ DeepEncoder 사전 학습  

| 항목 | 설정 |
|------|------|
| **목표** | 이미지 → 비전 토큰(압축) 변환, 고해상도 입력에서도 낮은 활성화 메모리 유지 |
| **데이터** | OCR 1.0 + OCR 2.0 (총 43 M 페이지) + LAION‑400M 샘플 중 100 M 개<sup>1</sup> |
| **배치 크기** | 1 280 |
| **학습 epoch** | 2 |
| **시퀀스 길이** | 4 096 토큰 |
| **Optimizer** | AdamW (β₁=0.9, β₂=0.999) |
| **학습률** | 5 × 10⁻⁵, **Cosine Annealing** 스케줄 사용 |
| **특징** | SAM‑base(80 M) + 16× 토큰 압축기 + CLIP‑large(300 M) 구조를 그대로 사용, 파라미터는 **380 M** |

> **핵심**: 16× 다운샘플링 컨볼루션(커널 3, 스트라이드 2, 패딩 1, 채널 256→1024)으로 4 096 → 256 토큰으로 압축하고, 윈도우‑어텐션이 활성화 메모리를 제한한다.

### 2️⃣ DeepSeek‑OCR (End‑to‑End) 학습  

| 항목 | 설정 |
|------|------|
| **전체 모델** | DeepEncoder (380 M) + DeepSeek‑3B‑MoE (3 B, 활성 파라미터 570 M) |
| **데이터 구성** | OCR 1.0 (70 %), OCR 2.0 (10 %), 일반 비전 데이터 (20 %) |
| **텍스트‑전용 데이터** | 전체의 10 % (길이 8 192 토큰, 모델 시퀀스 길이와 동일) |
| **GPU 클러스터** | 20 노드 × 8 A100‑40G (총 160 GPU) |
| **Pipeline Parallelism (PP)** | 4 단계: <br>PP0 – SAM + compressor (고정) <br>PP1 – CLIP (미고정) <br>PP2 / PP3 – 각각 6 레이어씩 MoE 디코더 |
| **Data Parallelism (DP)** | 40 배 (전역 배치 크기 640) |
| **Optimizer** | AdamW |
| **Learning‑rate** | 초기 3 × 10⁻⁵, **step‑based scheduler** 사용 |
| **멀티모달 훈련 속도** | 70 B 토큰/일 |
| **텍스트‑전용 훈련 속도** | 90 B 토큰/일 |
| **학습 단계** | ① DeepEncoder 사전 학습 → ② 전체 파이프라인 학습 → ③ Gundam‑Master 모드 (추가 6 M 샘플) 파인‑튜닝 |

> **주의**: DeepEncoder의 SAM‑base와 압축기 파라미터는 **freeze** 되어 있으며, CLIP 파트와 MoE 디코더만 업데이트된다. 이는 고해상도 시각 정보 손실을 최소화하면서 언어 모델 파라미터에 집중할 수 있게 한다.

### 3️⃣ 다중 해상도 모드 학습  

- **Native 모드**: Tiny(512×512, 64 토큰), Small(640×640, 100 토큰), Base(1024×1024, 256 토큰), Large(1280×1280, 400 토큰)  
- **Dynamic 모드**: Gundam (n × 640×640 타일 + 1024×1024 전역 view) 및 Gundam‑Master (1024×1024 + 1280×1280)  
- **학습 방식**: 위 모든 모드를 **동시** 학습시켜 하나의 모델이 다양한 해상도와 토큰 수를 처리하도록 함. 타일 수 *n* 은 2~9 사이에서 랜덤 샘플링된다.

> **수식** (유효 토큰 수)  
\[
N_{valid}= \Big\lceil N_{actual}\times\Big[1-\frac{\max(w,h)-\min(w,h)}{\max(w,h)}\Big]\Big\rceil
\]

### 4️⃣ 손실 함수 및 라벨링 전략  

| 라벨 유형 | 설명 |
|----------|------|
| **Coarse Annotation** | 전체 페이지 텍스트를 그대로 추출 (OCR 1.0) – Prompt `"Free OCR."` 로 제어 |
| **Fine Annotation** | 레이아웃 + 텍스트 좌표 (정규화 0‑1000) – Prompt `"Parse the document with layout."` |
| **멀티모달 라벨** | 차트 → HTML‑table, 화학식 → SMILES, 기하학 → 딕셔너리 구조 등 – 각각 전용 프롬프트 사용 |
| **텍스트‑전용 라벨** | 순수 텍스트 시퀀스 – 일반 언어 모델 학습과 동일한 크로스‑엔트로피 손실 적용 |

각 라벨은 **프롬프트 토큰**을 앞에 붙여 모델이 출력 포맷을 선택하도록 설계하였다.

### 5️⃣ 학습 결과 요약 (핵심 지표)

| Compression Ratio (텍스트 ÷ 비전) | Vision Tokens | OCR Precision |
|-----------------------------------|---------------|---------------|
| 10.5 × (600‑700 토큰) | 64 | 96.5 % |
| 6.7 × (600‑700 토큰) | 100 | 98.5 % |
| 15.1 × (900‑1000 토큰) | 64 | 85.9 % |
| 20 × (1200‑1300 토큰) | 64 | 59.1 % |
| 12 × (1000‑1100 토큰) | 100 | 91.5 % |

> **참고**: 위 수치는 Fox [21] 벤치마크 내 100 페이지(600‑1300 텍스트 토큰)에서 측정된 값이며, 실제 포맷 차이(레이아웃 어노테이션 포함) 때문에 실제 정확도는 더 높을 수 있다.
